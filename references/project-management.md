# 项目流程管理

## 何时阅读

- 要把用户需求落成一个 Mangou 项目
- 要拆分资产、分镜、生成任务和交付物
- 要检查项目是否还能继续推进

## 核心原则

1. 本 skill 只定义流程框架，不保存用户偏好、镜头审美或项目个性化经验。
2. 具体风格、角色偏好、镜头语言和审美判断应来自当前用户输入、项目 YAML、参考素材或运行时记忆。
3. 项目真相源是 `<workspace>/projects/<project_id>/` 下的 `project.json`、YAML、`tasks.jsonl` 和产物文件。
4. 飞书文档、聊天记录和外部素材只作为输入/输出通道，不替代项目真相源。

## 标准流程

1. 明确项目 ID、目标产物、输入素材和验收口径。
2. 初始化或定位项目目录。
3. 将需求拆为资产定义、分镜定义和生成任务。
4. 先补齐必要资产，再执行分镜图片、视频或拼接。
5. 每次执行后读取 `tasks.jsonl` 和 YAML `latest`，确认真实状态。
6. 将产物路径、失败原因或下一步动作回传给用户。

## 分镜管理

1. 一个 `storyboards/*.yaml` 表示一个物理镜头或一个清晰的逻辑叙事单元。
2. `content.story` 保留用户或剧本原文，不要为了简短丢失剧情约束。
3. `content.action`、`content.scene`、`content.duration` 应描述可执行信息，避免只写抽象审美词。
4. 连续镜头需要一致性时，优先引用上一镜产物或资产基准图，而不是重复堆叠 prompt。

## 资产管理

1. 角色、场景、道具等可复用视觉元素应先写入 `asset_defs/`。
2. 资产 YAML 只描述稳定身份和生成任务；不要写入临时审美偏好或会话结论。
3. 项目级风格规则如果需要持久化，应写入当前项目文件，而不是写回 skill 文档。

## 复刻与参考素材

复刻、临摹或参考已有片段时，按需执行：

1. 记录参考素材来源和用途。
2. 将参考素材拆成时间点、关键帧或视觉约束。
3. 把这些约束写入当前项目 YAML 或任务 prompt。
4. 不要把某次复刻经验沉淀为 skill 默认规则。

## 质量检查

检查顺序：

1. 项目目录是否正确。
2. YAML 字段是否完整。
3. provider 参数是否符合对应 reference。
4. `tasks.jsonl` 最新状态是否成功。
5. 产物路径是否可在项目目录内解析。
6. 结果是否满足当前用户需求和当前项目约束。
