# 一致性框架

## 何时阅读

- 多镜头需要保持角色、场景、道具或空间关系一致
- grid 母图需要切分为子镜
- 生成结果与上一镜、资产或剧情约束不一致

## 通用原则

1. 一致性约束来自当前项目，不来自 skill 默认审美。
2. 优先使用可验证的引用：资产基准图、上一镜产物、参考素材和 YAML 字段。
3. Prompt 中的连续性描述只作为辅助，不能替代 `params.images`、`image_urls` 或任务记录。
4. 不同项目可以采用不同的连续性策略，不把单个项目经验写成全局规则。

## Grid

1. `meta.grid` 是 grid 尺寸真相源。
2. grid 生成后如果需要独立镜头，运行 `storyboard split` 做物理切分。
3. 子镜声明了 `meta.grid_index` 时，回填必须优先按这个 1-based 索引。
4. 只有没写 `meta.grid_index` 时，才退回按 `sequence` 和文件名排序映射。
5. dense grid 或复杂 grid 先确认切分策略，再执行生成和回填。

## 连续镜头

1. 需要视觉继承时，把上一镜产物放进下一镜的媒体输入字段。
2. 需要资产稳定时，把资产基准图放进分镜的媒体输入字段。
3. 需要保持空间关系时，在当前 YAML 中明确站位、方向、前后关系或镜头运动。
4. 如果剧情要求跳切、风格变化或时空变化，应在 YAML 中显式说明，避免被一致性规则误修正。

## 诊断顺序

1. 检查当前镜头 YAML 是否包含必要角色、场景、动作和媒体引用。
2. 检查上一镜或资产产物路径是否存在。
3. 检查 `tasks.jsonl` 是否记录了正确的输入文件。
4. 检查 provider 是否支持当前媒体输入形式。
5. 根据失败类型只对当前任务补充约束，不扩大为 skill 默认规则。
